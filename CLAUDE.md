# CLAUDE.md — VOICE CONDUCTOR

## 一言で

楽器を弾けない人が、**右手の人差し指一本でジャズのソロを歌い**、**余った左手のジェスチャーでバンドを指揮する**、単一HTMLのコンテンポラリージャズ演奏アプリ。Kurt Rosenwinkel のバンドのような、有機的でスリリングなサウンドを目指す。

奏者は音符を選ばない。**表現の輪郭**だけを描き、ハーモニーエンジンが「絶対に外さない」着地を保証する。バンドは命令に従うのではなく、奏者の意思を**解釈し、反応し、時に逸脱する**。

---

## 絶対制約（NON-NEGOTIABLE）

1. **単一HTMLファイル。** 外部依存ゼロ。ビルドステップなし。`index.html` をブラウザで開けば動く。Web Audio API と Canvas のみ。CDN も使わない。
2. **モバイル縦持ち前提。** タッチ操作が主。マウスはフォールバック。右手が画面右、左手が画面左下を担う two-thumbs / two-finger 設計。
3. **発音レイテンシを妥協しない。** 指が触れた瞬間に鳴る。次の拍を待たせる発音クオンタイズは禁止。リズムの回収は背後のグルーヴに任せる（後述）。
4. **音を外す可能性をゼロに。** 右手の入力は常にその瞬間のアヴェイラブルノートにスナップ。どこを触っても和声的に成立する。
5. **localStorage / sessionStorage を使わない。** 状態はすべてメモリ内（JS変数）。
6. **何もしなくても曲が鳴っている。** 起動直後からバンドが自走。指を離しても破綻しない。「自分のせいで曲が止まった」体験を絶対に作らない。

---

## アーキテクチャ概観

3つの主体の三角関係でサウンドが生まれる:

```
┌─────────────┐     intensity      ┌──────────────────┐
│  右手(声)   │ ─────────────────> │                  │
│ 1本指リード │                    │  自律バンド      │
└─────────────┘                    │  エンジン        │
                                    │ (解釈・反応・逸脱)│
┌─────────────┐    示唆(suggest)   │                  │
│ 左手(意思)  │ ─────────────────> │                  │
│ 4ジェスチャー│                    └──────────────────┘
└─────────────┘                            │
                                           v
                                    ┌──────────────┐
                                    │ Web Audio出力 │
                                    └──────────────┘
```

推奨モジュール分割（すべて同一ファイル内のクラス／オブジェクトで）:

- `AudioEngine` — AudioContext、マスターバス、Rosenwinkel質感のエフェクトチェーン
- `Transport` — テンポ、拍、小節、look-aheadスケジューラ（25ms間隔でWeb Audio先読み）
- `HarmonyEngine` — 進行プール、現在のアヴェイラブルノート集合、転調予約の管理
- `RhythmSection` — ベース / ドラム / コンピングの自走と密度制御
- `RightHand` — リード入力 → ピッチ写像 → スナップ → 発音 → intensity算出
- `LeftHand` — ジェスチャー認識（速度で離散/連続を判定）→ バンドへの示唆
- `BandBrain` — 示唆と intensity を解釈し、反応・逸脱を決定する中枢
- `Visuals` — Canvas描画。縁の発光、靄、リードの軌跡、オンボーディングのゴースト

---

## 右手 — 声（1本指リード）

- **縦軸 = 音域。** 上が高音、下が低音。
- **横の動き = フレーズの前進感とアーティキュレーション。** 横位置そのものは音程に写像しない。指の上下のうねりがフレーズの形になる。
- **スナップ。** タッチY座標を、現在の `HarmonyEngine` のアヴェイラブルノート集合に写像。理想形では確率でクロマチックアプローチ／エンクロージャーを挿入して「人間っぽい揺らぎ」を足す（MVPなら確率を下げて安全側に）。
- **発音は即時。** タッチダウンで即発音。スライドで滑らかにピッチ移行（レガート/ポルタメント）。タッチアップで余韻を残して減衰。
- **intensity の算出。** 指の移動速度・到達音域の高さ・タッチ圧（取得できれば）から 0..1 の `leadIntensity` を毎フレーム算出し `BandBrain` へ流す。これがバンドの食いつきのトリガーになる。

## 左手 — 意思（4ジェスチャー語彙）

語彙は **4つだけ**。増やさない。画面左下のゾーンで操作。すべてジェスチャーで、常設UIは置かない（オンボーディングのゴーストを除く）。

| ジェスチャー | 種別 | 効果 |
|---|---|---|
| **縦スワイプ（ゆっくり連続）** | 連続値 | **温度。** 上で沸騰（密度・音域・歪み・シンバルレガート増）、下でクールダウン。最頻出。`temperature` 0..1 |
| **ホールド（押さえる／離す）** | 連続状態 | **間。** 押さえている間バンドが息を潜める（サスペンド）、離すと戻る。緊張と解放 |
| **上フリック（鋭い）／下フリック（鋭い）** | 離散イベント | 上=**キメ（バンドヒット）を放つ**。下=**ドロップ**（全員引いてリードだけ残す）。一回性の煽り |
| **横スワイプ** | 離散イベント | **進行を次へ差す。** 進行プールの切り替え（転調含む）。即時ではなく**次の区切りに予約して自然に滑り込む** |

### 誤爆対策 — ここが左手実装の本丸

縦スワイプ（温度）と上フリック（キメ）は方向が似ている。**速度で割る**:

- 指の移動の**加速度／ピーク速度**で判定。閾値を超える鋭い弾き = フリック（離散）。それ以下のゆるい移動 = 温度（連続）。
- フリックは方向（上/下）と発火を一度だけ。温度は指が触れている間の累積位置で連続更新。
- ホールドは「動かさず一定時間以上接地」で判定。微小なドリフトは許容。
- 横スワイプは横方向の支配的な変位で判定。縦成分が大きければ温度/フリック側に倒す。

判定パラメータ（速度閾値、ホールド時間、フリック最小変位）は**ファイル冒頭の CONFIG 定数にまとめ**、後から調律できるようにする。

### フィードバック（身体に覚えさせる）

ジェスチャーは目で確認できない。結果を**音と微かな視覚**で返す:

- 温度上昇中: 画面の縁がじわっと熱（暖色）を帯びる
- 間（ホールド中）: 画面が暗く沈み、リバーブテイルだけが残る
- キメ/ドロップ: 一瞬のフラッシュまたは沈降
- 横スワイプ予約中: 次の区切りまで、来たるべき調を示す淡いインジケータ

文字説明は出さない。結果が直感的に返ることで指が覚える。

## 自律バンドエンジン（BandBrain）— 理想形を最初から

バンドはスイッチ盤ではなく**指揮対象**。左手は命令ではなく**示唆（suggestion）**を出し、バンドが解釈して返す。3つの振る舞いを実装する:

### 1. 反応（reaction）— intensity センシング
左手で煽らなくても、右手のソロが高音域で激しくなれば（`leadIntensity` 上昇）、ドラムが食いつく・コンピングが密になる・ベースが歩き出す。**奏者の声そのものがバンドを動かす。**

### 2. 解釈（interpretation）— 示唆の出し方に応じた応答
左手の温度の**上げ方**を見る。急に上げれば急に応える。ためて上げれば焦らして返す（応答に意図的なラグとイージングを入れる）。同じ「温度+0.3」でも到達の仕方で結果が変わる。

### 3. 逸脱（deviation）— 確率的な逸れ
完全に従順なバンドはつまらない。低確率で**予測通りに反応しない瞬間**を入れる: 煽ったのに一拍ためる、間を作ったのにベースだけ残す、進行予約に対してリハーモナイズで応える、など。これが即興の会話を生む。逸脱は **musical に安全な範囲**で（HarmonyEngineの枠内）。確率は CONFIG で調律可能に。

`BandBrain` は毎区切り（小節 or フレーズ境界）で「現在の temperature / leadIntensity / 保留中の示唆 / 直近の逸脱履歴」を入力に、各パート（ベース・ドラム・コンピング）の次の振る舞いパラメータ（密度・レンジ・アクセント・レガート量）を決定する。

## ハーモニー（HarmonyEngine）— Rosenwinkel語法

- **進行プール**を複数持つ。少なくとも2系統: (a) モーダルで停滞感のある長尺ペダル＋上モノ変化、(b) Coltrane changes 的に急進する系統。横スワイプで行き来できると一気にあのバンドらしくなる。
- Lydian / altered dominant / メロディックマイナー由来のテンションを積極使用。
- アヴェイラブルノート集合は和音ごとに「コードトーン＋使えるテンション」で定義し、右手スナップとリード自動補完に供給。
- 転調予約は次のフレーズ境界で発火。ピボット的に滑り込ませる。

---

## Rosenwinkel の質感（AudioEngine）

- リードは**歪みの薄皮**（軽いサチュレーション/ソフトクリップ）と**リバーブの靄**。ノブで出すのではなく **`leadIntensity` と `temperature` に連動**させる。強く速くなぞると自然に前に出て歪む。
- リズムセクションは抑制的で空間的。シンバルレガート、ウォーキング/ペダルを行き来するベース、テンションを開離で置くコンピング。
- マスターに薄いコンプとリバーブセンド。靄はビジュアルの靄とも対応させる。

---

## ビジュアル方針（Visuals）

`frontend-design` スキルの精神に従い、テンプレ然とした見た目を避ける。

- 主役（signature）は**リードの軌跡と、温度に呼応する縁の発光**。Rosenwinkel の靄っぽい空気感をCanvasの粒子/グラデで。
- 演奏中はミニマル。情報を出しすぎない。色は楽器の世界から引く（暗い背景に、熱・靄・残響を色で表現）。
- **オンボーディングだけ別扱い。** 起動直後の数回だけ、左手の動きにゴーストの軌跡を重ねて「これが温度」と一瞬教える。数回触れたら消える。演奏中は完全ミニマル。

---

## 実装フェーズ（explore → plan → implement）

このアプリは「理想形を一気に」狙う方針だが、動く順に積む:

1. **Transport + AudioEngine + RhythmSection**: 何もせずバンドが自走して鳴る状態。まずこれが気持ちよく回ること。
2. **RightHand + HarmonyEngine**: 1本指で外さないリードが歌える。発音即時・スナップ・intensity算出。
3. **LeftHand**: 4ジェスチャー認識。誤爆対策（速度判定）を最優先で詰める。フィードバック視覚/音。
4. **BandBrain**: 反応 → 解釈 → 逸脱の順に。逸脱は最後、確率を絞って入れる。
5. **Visuals 仕上げ + オンボーディング**。

各フェーズで**実際に音を鳴らして確認**してから次へ。特に 1 と 2 の手触りが土台。

## コーディング規約

- 既存資産 **NIGHT LANGUAGE**（idiomorph アカウント、単一HTML / Web Audio / Canvas / auto-performer / リズムセクション）の構造とパターンを土台に流用する。あるなら参照して語法を踏襲。
- 調律対象の数値（速度閾値・確率・テンポ・密度カーブなど）は**ファイル冒頭の `CONFIG` オブジェクトに集約**。マジックナンバーをロジックに埋めない。
- Web Audio は **look-ahead スケジューラ**（`setInterval` 25ms で先読み、各音を `AudioContext.currentTime` 基準で予約）。`setTimeout` で個別発音しない。
- 関数・変数は音楽の言葉で命名（`leadIntensity`, `temperature`, `availableNotes`, `comping`, `walkBass`）。実装都合の名前にしない。
- コメントは「なぜ」を書く。特に誤爆判定と逸脱確率は意図を残す。
